# 研究方案 v0.1

## 题目暂定

**scRareRefine: a rare-aware refinement framework for single-cell annotation**

中文：

> **scRareRefine：面向稀有细胞类型的单细胞注释增强框架**

核心定位：

> 以 **scANVI** 作为基础注释模型，在其输出的 latent embedding、预测概率和不确定性基础上，引入类别均衡、原型距离、局部密度和 marker 一致性模块，对容易被大类吞并的稀有细胞类型进行选择性修正。

------

# 1. 研究背景

当前单细胞类型注释方法通常追求整体准确率，但在真实 scRNA-seq 数据中，细胞类型分布往往高度不均衡。大类细胞数量充足，模型容易学习；稀有类细胞样本少、表达信号弱，常被邻近大类吞并。结果是：

```text
overall accuracy 很高
但 rare-cell recall / rare-cell F1 很低
```

scANVI 是 scVI 的半监督扩展，适合做单细胞数据整合和细胞状态注释，能够输出 latent representation 和预测概率，因此非常适合作为基础 backbone。scANVI 原论文也明确用于跨数据集细胞类型标签转移和 probabilistic annotation。([Springer](https://link.springer.com/article/10.15252/msb.20209620?utm_source=chatgpt.com))

已有方法已经证明 rare-cell annotation / rare-cell detection 是实际问题。例如 scBalance 针对不平衡 scRNA-seq 数据设计 adaptive weighted sampling 和 sparse neural network，用于提升 rare cell type annotation。([Nature](https://www.nature.com/articles/s42003-023-04928-6?utm_source=chatgpt.com)) sc-SynO 则使用 synthetic oversampling 提升稀有类召回。([PMC](https://pmc.ncbi.nlm.nih.gov/articles/PMC8603509/?utm_source=chatgpt.com)) scCAD 和 GapClust 说明局部结构、cluster decomposition、local density 对 rare-cell identification 有价值。([Nature](https://www.nature.com/articles/s41467-024-51891-9?utm_source=chatgpt.com))

因此，本课题不是凭空造方向，而是把已有的 **类别不均衡学习、原型学习、局部密度检测、marker-based annotation** 模块裁缝式接到 scANVI 后面，形成一个轻量、可解释、可消融的 rare-aware annotation refinement framework。

------

# 2. 研究问题定义

## 2.1 主要任务

本课题主任务是：

> **rare known cell annotation refinement**

也就是：

```text
稀有类在训练集中存在
但样本数量少
模型容易把它们预测成邻近大类
```

这和 unknown cell detection 不同。

| 任务                   | 训练集中是否有该类 | 目标                          |
| ---------------------- | ------------------ | ----------------------------- |
| rare-cell annotation   | 有，但样本少       | 正确识别稀有已知类            |
| unknown-cell detection | 没有               | 判断该细胞不属于任何已知类    |
| annotation review      | 不一定             | 找出高风险/可能错误注释的细胞 |

本课题先聚焦 **rare known class**。这样指标更清楚，实验更容易闭环。

------

# 3. 核心假设

## H1：scANVI 的整体性能高，但 rare-class failure 会被 overall accuracy 掩盖

预期现象：

```text
overall accuracy 高
macro-F1 不一定高
rare-cell recall / rare-F1 偏低
```

## H2：类别均衡模块可以提升 rare-cell recall

通过 class-balanced / focal / weighted classifier，让稀有类在训练目标中获得更高权重。

## H3：原型距离和局部密度可以救回被大类吞并的 rare cells

如果某个细胞被 scANVI 预测为大类，但在 latent space 中更接近 rare-class prototype，且局部邻居支持 rare-class，则该细胞可能是 rare rescue candidate。

## H4：marker consistency 可以控制 rare rescue 的假阳性

rare-aware 模块容易提高 recall 但降低 precision。marker consistency verifier 的作用是确认 rescue 是否有生物学证据。

## H5：uncertainty gate 可以避免破坏原本正确的大类预测

只对高风险细胞进行 refinement，easy cells 保留 scANVI 原始预测。

------

# 4. 方法框架

## 4.1 总体流程

```text
Input: scRNA-seq expression matrix + reference labels
        ↓
Train scANVI
        ↓
Obtain:
  1. scANVI predicted label
  2. prediction probability
  3. entropy / margin uncertainty
  4. latent embedding z
        ↓
Rare-risk estimation
        ↓
Selective refinement:
  M1 balanced latent classifier
  M2 prototype-distance rescue
  M3 local-density / neighbor-purity rescue
  M4 marker-consistency verifier
        ↓
Output:
  1. final label
  2. rare-risk score
  3. rescue decision
  4. marker evidence
  5. review list
```

# 5. 模块设计

## M1. Balanced Latent Classifier

### 来源

| 来源                | 可借思想                                                     |
| ------------------- | ------------------------------------------------------------ |
| scBalance           | 针对不平衡 scRNA-seq annotation，用 adaptive weighted sampling 提升 rare-cell annotation。([Nature](https://www.nature.com/articles/s42003-023-04928-6?utm_source=chatgpt.com)) |
| Class-balanced loss | 按类别有效样本数调整 loss 权重                               |
| Focal loss          | 降低 easy samples 权重，强调 hard / minority samples         |

### 做法

先训练标准 scANVI，然后提取 latent embedding：

```text
z_i = scANVI latent representation
```

再在 latent space 上训练一个轻量分类器：

```text
p_balanced(y | z)
```

分类器可以选：

```text
weighted logistic regression
weighted MLP
class-balanced MLP
focal-loss MLP
```

第一版建议：

```text
weighted logistic regression / shallow MLP
```

不要一开始改 scANVI 内部 loss，工程风险低。

### 解决的问题

```text
大类样本多
→ scANVI 决策边界偏向大类
→ rare cells 被预测为 major class
```

### 预期提升指标

| 指标              | 预期变化                              |
| ----------------- | ------------------------------------- |
| rare-cell recall  | 提升                                  |
| macro-F1          | 提升                                  |
| balanced accuracy | 提升                                  |
| overall accuracy  | 可能基本保持，也可能小幅下降          |
| rare precision    | 不一定提升，需要 marker verifier 控制 |

------

## M2. Prototype-distance Rescue

### 来源

来自 prototypical networks / prototype learning。

核心思想：

```text
每个类别用 latent prototype 表示
细胞归属不仅看 softmax 概率，也看它距离哪个类别原型更近
```

### 做法

对每个类别建立 prototype：

```text
μ_c = mean(z_i), y_i = c
```

对每个细胞计算到各类 prototype 的距离：

```text
d_c(x) = distance(z_x, μ_c)
```

重点观察这种情况：

```text
scANVI 预测为 major class
但该细胞更接近 rare-class prototype
且 scANVI 置信度不高
```

这类细胞进入 rare rescue candidate。

### 解决的问题

```text
稀有细胞在 embedding 空间中靠近邻近大类
softmax 分类器倾向于输出大类
但几何结构上它可能更接近 rare class
```

### 预期提升指标

| 指标                    | 预期变化         |
| ----------------------- | ---------------- |
| rare-cell recall        | 提升             |
| rare-cell F1            | 提升             |
| per-class F1            | 稀有类提升更明显 |
| top-k review recall     | 提升             |
| confusion: rare → major | 降低             |

------

## M3. Local-density / Neighbor-purity Module

### 来源

| 来源     | 可借思想                                                     |
| -------- | ------------------------------------------------------------ |
| GapClust | 利用局部距离 gap / local structure 识别 rare cells           |
| scCAD    | 通过 cluster decomposition 和 anomaly detection 识别 rare cell types；论文在 25 个真实 scRNA-seq 数据集上 benchmark。([Nature](https://www.nature.com/articles/s41467-024-51891-9?utm_source=chatgpt.com)) |

### 做法

在 scANVI latent space 中构建 kNN graph，计算：

```text
local_density(x)
neighbor_label_distribution(x)
neighbor_purity(x)
rare_neighbor_ratio(x)
major_neighbor_ratio(x)
distance_to_rare_neighbors(x)
distance_to_major_neighbors(x)
```

典型规则：

```text
如果一个细胞被预测为 major class
但：
  1. 局部邻居中 rare-class support 较高
  2. 位于 major-class 低密度边缘
  3. 靠近 rare-class prototype
则标记为 rare rescue candidate
```

### 解决的问题

```text
稀有细胞不是完全离群
而是在大类附近形成小的局部亚群
普通分类器容易把它们吞并进大类
```

### 预期提升指标

| 指标                 | 预期变化   |
| -------------------- | ---------- |
| rare recall          | 提升       |
| rare-F1              | 提升       |
| top-k review recall  | 明显提升   |
| per-class F1         | 稀有类提升 |
| manual review burden | 降低       |

------

## M4. Marker-consistency Verifier

### 来源

| 来源           | 可借思想                                                     |
| -------------- | ------------------------------------------------------------ |
| ScType         | 基于 positive / negative marker combinations 进行 cell-type annotation。([Nature](https://www.nature.com/articles/s41467-022-28803-w?utm_source=chatgpt.com)) |
| CellAssign     | 使用先验 marker gene 信息进行 probabilistic cell-type assignment。([PubMed](https://pubmed.ncbi.nlm.nih.gov/31501550/?utm_source=chatgpt.com)) |
| CellMarker 2.0 | 人工整理的人和小鼠细胞类型 marker 数据库。([OUP Academic](https://academic.oup.com/nar/article/51/D1/D870/6775381?utm_source=chatgpt.com)) |
| PanglaoDB      | 整合 mouse / human 单细胞数据和 marker 信息的数据库。([PMC](https://pmc.ncbi.nlm.nih.gov/articles/PMC6450036/?utm_source=chatgpt.com)) |
| CIARA          | 用 marker localization 思想寻找 rare cell type markers       |

### 做法

marker verifier 不负责全局注释，而只验证 rescue 是否可信。

对每个 candidate rare cell，计算目标 rare class 的 marker score：

```text
marker_score(c)
= positive_marker_activity(c)
- negative_marker_activity(c)
```

可选实现方式：

```text
mean expression score
rank-based marker score
AUCell-style marker activity
positive / negative marker specificity score
```

判定逻辑：

```text
prototype/local-density 支持 rare class
且 marker score 支持 rare class
→ 接受 rescue

prototype/local-density 支持 rare class
但 marker score 不支持
→ 不直接改标签，放入 review list

marker score 支持 rare class
但几何证据弱
→ 放入 review list，不自动修改
```

### 解决的问题

```text
提高 rare recall 后，容易引入 false rare positives
```

marker verifier 用来控制：

```text
major cell 被错误 rescue 成 rare class
```

### 预期提升指标

| 指标                   | 预期变化 |
| ---------------------- | -------- |
| rare precision         | 提升     |
| rare-F1                | 提升     |
| false positive rate    | 下降     |
| top-k review precision | 提升     |
| 可解释性               | 明显增强 |

------

## M5. Uncertainty Gate / Selective Refinement

### 来源

来自 selective classification / uncertainty-aware annotation。popV 这类单细胞注释共识方法也强调输出 uncertainty scores，并把困难细胞群体交给人工审阅。([Nature](https://www.nature.com/articles/s41588-024-01993-3?utm_source=chatgpt.com))

### 做法

根据 scANVI 输出计算：

```text
max_prob
entropy
margin = p_top1 - p_top2
top2_is_rare
prototype_margin
local_density_score
```

进入 refinement 的条件可以是：

```text
entropy 高
或 max_prob 低
或 margin 小
或 top2 是 rare class
或 prototype/local-density 指向 rare class
```

最终策略：

```text
easy cells:
  保留 scANVI 原始预测

risky cells:
  进入 rare-aware refinement
```

### 解决的问题

```text
如果对所有细胞强行修正
会破坏原本正确的大类注释
```

### 预期提升指标

| 指标                     | 预期变化       |
| ------------------------ | -------------- |
| overall accuracy         | 尽量保持       |
| rare recall              | 提升           |
| rare-F1                  | 提升           |
| ECE / calibration error  | 可能改善       |
| top-k review recall      | 提升           |
| 修改率 modification rate | 控制在较低水平 |

------

## M6. Latent Oversampling，第二阶段可选

### 来源

sc-SynO 使用 LoRAS synthetic oversampling 增强 rare-cell annotation，目标是从已有 rare cells 生成 synthetic samples，用于缓解 minority-majority imbalance。([PMC](https://pmc.ncbi.nlm.nih.gov/articles/PMC8603509/?utm_source=chatgpt.com))

### 做法

不建议第一阶段在原始表达矩阵生成细胞，建议只在 scANVI latent space 做轻量增强：

```text
rare-class latent vectors
        ↓
SMOTE / LoRAS-like interpolation
        ↓
训练 balanced latent classifier
```

### 为什么放第二阶段

因为 oversampling 的常见风险是：

```text
rare recall 上升
rare precision 下降
```

所以它适合作为扩展消融，而不是第一版主线。

# 6. 数据集设计

## 6.1 数据集选择原则

本课题需要的不是单纯“大数据集”，而是满足以下条件的数据集：

| 条件                                 | 原因                           |
| ------------------------------------ | ------------------------------ |
| 有可靠 cell-type label               | 需要计算 rare recall / rare-F1 |
| 存在天然小类                         | 适合 rare-cell evaluation      |
| 最好有 batch / donor / protocol 信息 | 方便做跨批次、跨数据集验证     |
| 有 marker 可查                       | 支持 marker consistency        |
| 数据规模适中                         | 方便快速迭代                   |

# 8. Rare-class 构造方式

## 8.1 Natural rare setting

直接使用数据集中天然占比低的类别。

定义：

```text
rare class = proportion < 5%
```

这个阈值与 scBalance 对 minor population 的设定比较接近。

优点：

```text
真实
容易解释
```

缺点：

```text
某些数据集中 rare 类太少，训练不稳定
```

------

## 8.2 Controlled downsampling setting

人为把某个类别下采样成 rare class。

例如：

```text
rare training cells = 20 / 50 / 100 / 200
```

测试集保留真实分布。

优点：

```text
可以系统评估不同稀有程度
```

建议设置：

| Setting       | rare train size |
| ------------- | --------------- |
| extreme rare  | 20              |
| rare          | 50              |
| moderate rare | 100             |
| mild rare     | 200             |

这样可以画出：

```text
rare-cell F1 vs rare train size
```

这个图会很适合论文。

------

## 8.3 Near-major-class setting

选择与大类相似、容易混淆的 rare class。

判断方式：

```text
scANVI baseline confusion matrix
或 prototype distance
或 marker overlap
```

例如：

```text
ASDC vs cDC2 / pDC
epsilon cells vs endocrine subtypes
pDC vs DC / monocyte
```

优点：

```text
更贴近“被邻近大类吞并”的核心问题
```

# 9. 实验设计

## 9.1 Baseline

必须包含：

| Baseline                          | 作用                         |
| --------------------------------- | ---------------------------- |
| scANVI                            | 主 baseline                  |
| scANVI + normal latent classifier | 检查是不是简单分类器就能提升 |
| scANVI + weighted classifier      | 对应 M1                      |
| CellTypist，可选                  | 外部 annotation baseline     |
| scBalance，可选                   | rare-cell annotation 对照    |
| sc-SynO，可选                     | oversampling 对照            |

第一阶段最小 baseline：

```text
B0: scANVI
B1: scANVI + standard latent classifier
B2: scANVI + balanced latent classifier
```

------

## 9.2 Ablation study

建议按这个顺序做：

```text
B0 = scANVI
B1 = B0 + uncertainty gate
B2 = B1 + balanced latent classifier
B3 = B2 + prototype-distance rescue
B4 = B3 + local-density module
B5 = B4 + marker-consistency verifier
B6 = B5 + latent oversampling，可选
```

更合理的论文表格可以写成：

| Model             | Gate | Balanced | Prototype | Density | Marker | Rare-F1 | Macro-F1 | Overall Acc |
| ----------------- | ---- | -------- | --------- | ------- | ------ | ------- | -------- | ----------- |
| scANVI            | -    | -        | -         | -       | -      |         |          |             |

# 10. 指标体系

## 10.1 主指标

| 指标                | 作用                        |
| ------------------- | --------------------------- |
| rare-cell recall    | 是否救回稀有类              |
| rare-cell precision | 是否引入大量 false positive |
| rare-cell F1        | rare 类综合表现             |
| macro-F1            | 避免大类主导                |
| balanced accuracy   | 类别不均衡下更公平          |
| per-class F1        | 展示每个细胞类型改善        |

------

## 10.2 辅助指标

| 指标                            | 作用                                  |
| ------------------------------- | ------------------------------------- |
| overall accuracy                | 证明没有明显损害整体性能              |
| confusion matrix                | 看 rare class 被哪些大类吞并          |
| rare-to-major error rate        | 直接量化 rare 被大类吞并              |
| major-to-rare false rescue rate | 看 rescue 是否过度                    |
| modification rate               | 有多少细胞被修改                      |
| ECE / calibration error         | 看置信度是否更可靠                    |
| top-k review recall             | 审阅前 k% 可疑细胞覆盖多少 rare/error |

------

## 10.3 top-k review recall

这个指标很适合你的框架。

定义：

```text
把细胞按 rare-risk score 从高到低排序
取前 k%
看其中覆盖了多少 rare-cell errors 或真实 rare cells
```

例如：

| k       | 指标                  |
| ------- | --------------------- |
| top 1%  | top-1% review recall  |
| top 5%  | top-5% review recall  |
| top 10% | top-10% review recall |

这个指标可以支撑一个应用场景：

> scRareRefine 不仅自动修正一部分 rare cells，也能给实验人员提供高价值审阅列表。

# 11. 预期结果

## 11.1 理想结果形态

在 scANVI baseline 上，full model 应该表现为：

| 指标                | 预期                            |
| ------------------- | ------------------------------- |
| overall accuracy    | 基本不下降，或下降很小          |
| macro-F1            | 提升                            |
| rare recall         | 明显提升                        |
| rare precision      | 通过 marker verifier 保持或提升 |
| rare-F1             | 提升                            |
| top-k review recall | 明显提升                        |
| per-class F1        | 稀有类提升最明显                |

## 11.2 你要避免的结果

如果出现：

```text
rare recall 提升
但 rare precision 大幅下降
overall accuracy 明显下降
```

说明模型在“过度 rescue”。

解决办法：

```text
提高 gate 阈值
加强 marker verifier
降低 rare rescue 权重
只输出 review，不直接改 label
```

# 12. 论文故事线

## 12.1 核心问题

> Existing single-cell annotation models can achieve high overall accuracy, but their performance on rare cell types remains vulnerable to class imbalance and local embedding absorption by neighboring major classes.

中文：

> 现有单细胞注释模型整体准确率较高，但在稀有细胞类型上仍容易受到类别不均衡和邻近大类吞并效应的影响。

------

## 12.2 方法主张

> We propose scRareRefine, a rare-aware annotation refinement framework built on top of scANVI. It selectively refines high-risk cells using balanced latent classification, prototype-distance rescue, local-density evidence, and marker-consistency verification.

中文：

> 本研究提出 scRareRefine，一个基于 scANVI 的稀有细胞注释增强框架。该框架只对高风险细胞进行选择性修正，并综合类别均衡、原型距离、局部密度和 marker 一致性证据。

------

## 12.3 创新点

| 创新点                         | 对应问题                           | 对应指标                            |
| ------------------------------ | ---------------------------------- | ----------------------------------- |
| rare-aware refinement          | overall accuracy 掩盖 rare failure | rare-F1、macro-F1                   |
| geometry-aware rescue          | rare cells 被大类吞并              | rare recall、top-k review recall    |
| marker-consistent verification | rare rescue 假阳性                 | rare precision、false rescue rate   |
| selective correction           | 避免破坏 easy cells                | overall accuracy、modification rate |

# 16. 风险与应对

| 风险                              | 表现                          | 应对                                                         |
| --------------------------------- | ----------------------------- | ------------------------------------------------------------ |
| rare recall 提升但 precision 下降 | rescue 过度                   | 加强 marker verifier，调高 gate 阈值                         |
| marker 数据库不完整               | 某些 rare class 无可靠 marker | 同时使用 training-set DE markers，但只能从 train set 计算，避免泄漏 |
| rare class 太少                   | prototype 不稳定              | 设置最小样本数；使用 shrinkage prototype 或合并相近类        |
| scANVI baseline 已经很强          | 提升不明显                    | 重点报告 rare class、top-k review、controlled downsampling   |
| 数据集标签粒度不同                | 跨数据集标签不一致            | 先做同一数据集内 downsampling，再做跨数据集验证              |
| 模块太多显得拼凑                  | 审稿人质疑创新性              | 强调 selective rare-aware refinement，而不是简单 ensemble    |

------

# 17. 当前最建议的执行顺序

你的第一步不是马上写完整代码，而是先做：

```text
数据集审计 + scANVI rare failure baseline
```

也就是先回答：

1. 哪些类是 rare class？
2. scANVI 在这些类上的 recall / F1 是否真的低？
3. rare cells 主要被错分到哪些 major class？
4. 这些错误细胞是否有高 entropy / 小 margin？
5. 它们在 latent space 是否接近 rare prototype？
6. marker 是否能支持修正？

如果这 6 个问题能跑通，scRareRefine 这个方向就基本成立。

# 18. 一句话版方案

> 本研究以 scANVI 为基础模型，针对单细胞注释中稀有细胞类型容易被大类吞并的问题，提出 scRareRefine：一个结合不确定性门控、类别均衡 latent 分类器、原型距离、局部密度和 marker 一致性验证的稀有细胞注释增强框架。该框架旨在不显著损害整体准确率的前提下，提高 rare-cell recall、rare-cell F1、macro-F1 和 top-k review recall，并为稀有类修正提供可解释的 marker 证据。
